Capítulo 2

Bootstrap é um framework de desenvolvimento para o web frontend. Com foco em mobile first, é uma excelente ferramenta para agilizar o desenvolvimento e manter a responsividade, na medida que os templates da ferramentas são programadas para se adaptar, primeiramente, aos mobiles e aumentar o viewport até que encontre o correto.

Logo framework Bootstrap

Beleza mas onde ele é usado

Os principais motivos de se utilizar esse framework é a facilidade de se utilizar e aprender, responsividade que a ferramenta proporciona a página, permite a customização. Com a soma desses fatores, o desenvolvimento se torna mais ágil se adequando a metodologias como scrum, se enquadrando bem as necessidades atuais do mercado de TI.

Como utilizar bootstrap

Ler a documentação https://getbootstrap.com/docs/5.3/getting-started/introduction/ De forma sucinta, copiar os links que contém os arquivos de CSS e JS da ferramenta e inseri-los no no documento Html conforme indicado e explicado de maneira mais profunda pela documentação.

Sistema Grid do Bootstrap

o sistema de grid do Bootstrap é formado por linhas e colunas que devem estar dentro de um container(DIV). A soma da coluna por linha nunca pode ultrapassar 12. As colunas comportam os mais diversos conteúdos, vídeos, textos e imagens. A declaração da coluna deve respeitar o tamanho da tela. xs - smartphones sm - tablets md - desktops lg - tv’s … por padrão, para páginas que serão vistas em várias telas como streamings e etc… é adotado o md, utilizando o css para auxiliar a transição entre outros tamanho de tela exemplo: código

Container fixed x Container Fluid

O Container é o box base para a montagem do layout com o Bootstrap. Todos os outros elementos devem estar dentro do container. Um container é uma div que foi declarada com a classe específica de container do Bootstrap e é assim que vamos trabalhar, utilizando suas classes já pré definidas, realizando alguns ajustes quando necessário. segue link de classes predefinidas do bootstrap: bootstrap components Existem duas classes de containers: o tipo Fixed e Fluid. O container fixed mantém o tamanho em caso de redimensionamento da tela e fica posicionado no centro da viewport. O container-fluid tem a característica inversa do fixed, porque ele se expande e toma todo o espaço da viewport.
