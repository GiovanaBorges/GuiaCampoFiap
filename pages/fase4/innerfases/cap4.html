Github e Git

Feito por : Arthur
Capítulo 4

O GitHub é uma famosa rede social de compartilhamentos de código, é um repositório de códigos-fontes, os quais podem ser privados ou abertos, de desenvolvedores iniciantes e até grandes empresas. O GitHub foi criado em 2008 e, dez anos mais tarde, foi comprado pela Microsoft.

O Git é um software de versionamento, ou sistema de controle de versão. Ele é responsável por registrar as alterações feitas no projeto, comitando em versões, onde cada ponto de versionamento representa uma versão do projeto.

Logo do Git e Github

pra que serve git e github

Além de ser uma rede social facilitando o contato entre desenvolvedores, é um repositório digital para códigos fontes, podendo ser utilizado para versionar, armazenar e compartilhar seu código.

por que usar git ?

Todos os projetos passam por versões, mesmo depois de concluídos. É importante utilizar desde o começo uma ferramenta que auxilie a controlar as alterações feitas no código, viabilizando um maior controle, por meio de ferramentas auxiliares como as branchs e descrições dos commits.

Básico Git

Para instalar, você pode acessar o site do Git: .

Com o Git instalado em sua máquina, chegou a hora de nos apresentarmos para ele, ou seja, configurar o sistema para ele identificar o desenvolvedor responsável pelas alterações.

Para criar um diretório, você deverá criar uma pasta (pode ser no desktop mesmo) e, com o botão direito, clicar na opção Git Bash Here:

Windows explorer

Após clicar em Git Bash Here, o terminal se abrirá e você digitará o primeiro comando: git init.

Imagem git bash terminal

Após o comando git init, você notará uma pasta oculta chamada .git, isso quer dizer que iniciamos o nosso repositório, mas precisamos configurá-lo com o nome e o e-mail.Para isso, basta digitar o seguinte comando no terminal:

git config --global user.name "Nome de usuário"

git config --global user.email "email@dominio.com.br"

Para verificar se o nome e e-mail foram setados corretamente, basta entrar com os comandos:

git config user.name

git config user.email

Estado dos arquivos e commits

Em sua pasta Projeto Git, crie um arquivo index.txt.

Agora, em seu terminal, execute o comando: git status, ele informará o status.

a mensagem untracked significa que o git desconhece o arquivo, isto é, o arquivo precisa ser adicionado Basta utilizar um git add: nome do arquivo. Além disso, utilizar um git commit e adicionar -m”” caso queira utilizar um comentário
